# monthly-inbox-bot : Philosophy

## Purpose
- 月報内容をもとに、フィードバック文の草案作成を支援する
- 人が判断・調整する前提で、AIは補助に徹する

## Usage Assumption
- 月1回程度の実行を想定
- リアルタイム性は不要
- 再現性・説明可能性を重視

## Design Principles
- プロンプトは人が管理し、コードから変更しない
- 処理は stage 単位で分離し、月次で完結させる
- 入力・中間・出力は必ず分離・隔離する
- ログは「後から人に説明できる」ことを目的に残す

## Non-goals
- 完全自動評価・人事判断の代替はしない
- 常時稼働・リアルタイム分析は行わない
